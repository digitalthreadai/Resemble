Setup Guide

Get Resemble running on web, iOS, and Android

1
Prerequisites
~2 min

Ensure the following tools are installed on your machine before proceeding.

Required

  • Node.js 18+ — nodejs.org
  • npm (bundled with Node.js)
  • Git — git-scm.com

Optional (for native builds)

  • Expo CLI — npm i -g expo-cli
  • EAS CLI — npm i -g eas-cli
  • Android Studio — for Android emulator
  • Apple Developer Account — for iOS distribution
2
Clone & Install
~3 min

Clone the repository and install all dependencies.

$ git clone https://github.com/digitalthreadai/Resemble.git
$ cd Resemble
$ npm install
Note: The install may take a minute as native camera and ML dependencies are resolved.
3
Environment Variables
~1 min

Resemble requires minimal environment configuration. The only variable is needed for celebrity match mode.

VariableDescriptionRequired
EXPO_PUBLIC_GROQ_API_KEY Groq API key for celebrity search Only for celebrity mode 
$ echo "EXPO_PUBLIC_GROQ_API_KEY=your_key" > .env.local
4
Run Dev Server
~2 min

Choose a platform to start the development server.

npm run web

Opens in your default browser at http://localhost:4003

Fastest way to test UI changes. Camera uses the browser's getUserMedia API.

npm run android

Launches the Android emulator or connects to a USB device via ADB.

Requires Android Studio with an emulator image configured.

npm run ios

Launches the iOS Simulator (macOS only).

Requires Xcode and a valid simulator runtime. Not available on Windows.

npx expo start

Starts the Expo dev server with a QR code. Scan with Expo Go on a physical device or press w for web, a for Android, i for iOS.

5
EAS Cloud Build
~10 min

Use Expo Application Services (EAS) to build native binaries in the cloud.

# Install EAS CLI and log in
$ npm i -g eas-cli
$ eas login

# Build for iOS
$ eas build --platform ios --profile preview

# Build for Android
$ eas build --platform android --profile preview

Build Profiles

ProfilePurposeOutput
developmentDev client with debug tools.apk / .app
previewInternal testing builds.apk / .ipa (ad hoc)
productionStore-ready builds.aab / .ipa (App Store)

7
Face Detection Models

Web

Models are auto-loaded from CDN via face-api.js on first page load. Three models are fetched: SSD MobileNet (face detection), 68-point landmark model, and face recognition net. Total download is approximately 50 MB, cached by the browser after first load.

Native (iOS / Android)

MobileFaceNet is bundled with the app binary at 5.2 MB. Google ML Kit Face Detection is included via the Expo SDK and handles real-time 30fps landmark extraction through Vision Camera frame processors.

8
Deployment

GitHub Pages (web)

$ npx expo export --platform web
$ npx gh-pages -d dist

Vercel / Netlify

Point the build command to npx expo export --platform web and set the output directory to dist. No server required — the export produces a fully static site.

App Store / Google Play

Use EAS Submit to upload production builds directly to the stores:

$ eas submit --platform ios
$ eas submit --platform android
9
Testing

Verify your setup with these commands.

# TypeScript type check
$ npx tsc --noEmit

# Web build verification
$ npx expo export --platform web

Both commands should exit with code 0. If the type check reports errors, fix them before proceeding with deployment.
